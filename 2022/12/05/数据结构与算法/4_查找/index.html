数据结构与算法-4_查找

4. 查找

夜阑月明人尽望，铁骑刀锋梦中伏。

4.1 概念

  1. 查找：在数据集合中寻找满足某种条件的数据元素的过程。

  2. 查找表：用于查找的数据集合。

    • 静态查找表：只读的查找表。常用顺序查找、二分查找、散列查找。

    • 动态查找表：需要动态插入或删除的查找表。常用二叉排序树、散列查找。

  3. 关键字：数据元素中唯一标识该元素的某个数据项的值。

    查找长度：查找需要比较关键字次数。

    平均查找长度，ASL：所有查找过程中查找长度的平均值。

4.2 查找算法

线性查找

  1. 顺序查找：适用于顺序表和链表。
    • 对无序线性表，查找成功的平均长度为\(\frac{n+1}2\)，查找失败的平均长度为\(n+1\)。
    • 对有序线性表，可以通过比较来提前得知查找失败。
  2. 二分查找：仅适用于有序顺序表，不适用于链表。
    • 查找成功的平均长度为\(\frac{n+1}{n}\log_2(n+1)-1\)。
  3. 分块查找：又称索引顺序查找。
    • 将查找表分为多个块内元素可以无序，块间有序的查找块。
    • 建立索引表，记录每个块的最大值和第一个元素的地址。索引有序排列。
    • 先查索引，后查块。

树形查找

  1. 二叉排序树

    • 查找：从根结点开始逐层向下比较。
      • 查找效率：取决于树的高度。平衡时为\(O(log_2n)\)，极不平衡时为\(O(n)\)。
    • 插入：递归
      1. 若原二叉排序树为空，直接插入。
      2. 若关键字小于根节点，插入左子树，否则插入右子树。
    • 构造：从空树开始依次插入元素。
    • 删除：
      • 若为叶结点，直接删除。
      • 若只有一棵左子树或右子树，则让子树替代该结点位置。
      • 若有左右两棵子树，则让其后继或前驱替代其位置，再删除其后继或前驱的原本结点。

  2. 二叉平衡树

    • 结点的平衡因子：左子树和右子树的高度差，只可以是\(-1,0,1\)。

    • 插入：按二叉排序树的方式插入，检查插入路径上的结点，对以离插入结点最近的，平衡因子绝对值大于0的结点为根的子树（最小不平衡子树）进行调整。

      • LL平衡旋转（右旋）：

        • 原因：结点A左子结点B的左子树插入结点，A的平衡因子由1变成2。

        • 过程：B取代A，A成为B的右子结点，B原本的右子树成为A的左子树。

      • RR平衡选择（左旋）：

        • 原因：结点A右子结点B的右子树插入结点，A的平衡因子由-1变成-2。

        • 过程：B取代A，A成为B的左子结点，B原本的左子树成为A的右子树。

      • LR平衡旋转（先左后右双旋）：

        • 原因：结点A左子结点B的右子树插入结点，A的平衡因子由1变成2。
        • 过程：先将B的右子结点C左旋到B，再将C右旋到A。

      • RL平衡旋转（先右后左双旋）：

        • 原因：结点A右子结点B的左子树插入结点，A的平衡因子由-1变成-2。
        • 过程：先将B的左子结点C右旋到B，再将C左旋到A。

    • 构造：从空树开始依次插入元素。

    • 删除：按二叉排序树的方式删除。找到最小不平衡子树进行调整。然后向根节点逐层回溯检查。

  3. 红黑树：满足一定红黑性质的二叉排序树。

    • 红黑性质：
      • 每个结点是红的或黑的。
      • 根结点是黑的。
      • 叶结点都是黑的。引入\(n+1\)个外部叶结点，使得每个内部结点的左右子结点非空。
      • 不存在两个相邻的红结点。
      • 对每个结点，从该结点到任意一个叶结点的简单路径上，所含黑结点的数量相同。
    • 红黑树的性质：
      • 从根结点到叶结点的最长路径不大于最短路径的2倍。
      • 有\(n\)个内部结点的红黑树的高度\(h\le2\log_2(n+1)\)。
    • 插入：
      1. 按二叉排序树的方式插入，并设该结点为红色。
      2. 若其为根节点，则将其改为黑色，结束。
      3. 若其父结点为黑色，结束。
      4. 若其叔结点为红色，则将父结点和叔结点都改为黑色，将爷结点改为红色。然后将爷结点作为新结点进行循环调整。
      5. 其叔结点为黑色：
        • 该结点为爷结点的左子结点的左子结点：LL，并交换父结点和爷结点的颜色
        • 该结点为爷结点的左子结点的右子结点：LR，并交换父结点和爷结点的颜色
        • 该结点为爷结点的右子结点的左子结点：RL，并交换父结点和爷结点的颜色
        • 该结点为爷结点的右子结点的右子结点：RR，并交换父结点和爷结点的颜色
    • 删除

  4. B树：\(m\)阶B树是所有结点的平衡因子均等于0的\(m\)路平衡查找树。

    • m阶B树满足的特性：
      • 树中每个结点至多有\(m\)棵子树，即至多有\(m-1\)个关键字。
      • 非叶根结点至少有2棵子树，即至少有1个关键字。
      • 除根结点以外的所有非叶结点至少有\([\frac m2]\)棵子树，即至少有\([\frac m2]-1\)个关键字。
      • 非叶结点结构：\(n,P_0,K_1,P_1,....K_n,P_n\)，\(K_i\)为关键字，\(P_i\)为指针。
      • 所有叶结点都是外部结点，在同一层，实际不存在。
    • 查找：多路比较关键字。
    • 高度：对一棵有\(n\)个关键字，高度为\(h\)、阶数为\(m\)的B树：
      • \(n\le m^h-1,h\ge\log_m(n+1)\)
      • \(h\le\log_{[\frac m2-1]}\frac{n+1}2+1\)
    • 插入：定位和插入，如果结点关键字多于\(m-1\)，则需要进行分裂。
      • 分裂：\([\frac m2]\)位置的关键字插入父结点，其右的关键字作为新结点。
    • 删除：定位和删除，如果结点关键字少于\([\frac m2]-1\)，则需要进行合并。
      • 合并：如果兄弟结点关键字多于\([\frac m2]-1\)，则调整其关键字进入父结点，原父结点中关键字进入原结点。否则合并父结点中关键字作为新结点。

  5. B+树

    • m阶B+树满足的特性：
      • 树中每个分支结点至多有\(m\)棵子树。
      • 非叶根结点至少有2棵子树，其他分支结点至少有\([\frac m2]\)棵子树。
      • 结点的子树个数和关键字个数相等。
      • 所有叶结点包含全部关键字并指向相应记录的指针。
      • 叶结点将关键字按大小排列并且相邻叶结点按大小顺序相互连接，支持顺序查找。
      • 所有分支结点仅包含其子结点中关键字的最大值和指向子结点的指针。

散列查找

  1. 散列函数：把查找表中的关键字映射为对应地址的函数。

    散列表：根据关键字直接进行访问的数据结构。

    • 散列函数定义域包含全部关键字，值域依赖于散列表大小。
    • 散列函数计算出的地址应尽可能均匀分布、减少冲突。
    • 散列函数应尽量简单，计算时间短。

  2. 常用散列函数：

    • 直接定址：取关键字的某个线性函数值\(H(x)=ax+b\)。
    • 除留余数：取一个不大于表长但接近表长的质数\(p\)，取\(H(x)=x\mod p\)
    • 数字分析：\(r\)进制数的\(r\)个数码在各位上出现的频率不相同，则选取数码分布较平均的位进行散列。
    • 平方取中：取关键字的平方值的中间几位。

  3. 处理冲突的方法：

    • 开放定址：空闲地址向非同义词开放。
      • 线性探测：冲突发生时，按序查看表中下一个空闲单元。
      • 平方探测：冲突发生时，按序查看相对原址\(1,-1,2^2,-2^2,...\)的下一个空闲单元。
      • 双散列：第一个散列函数冲突，则使用第二个散列函数。
      • 随机数
    • 拉链：同义词存于同一个链表中。

  4. 散列表的查找效率取决于：散列函数、处理冲突办法、装填因子

    • 装填因子=记录数/表长。
