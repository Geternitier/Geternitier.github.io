继承

机制：基于目标代码的复用

1. 继承

继承方式：public、private、protected

  • public继承：继承父类的public为public、继承protected为protected，私有不继承

  • protected继承：继承父类的public、protected为protected

  • private继承：继承父类的public、protected为private

派生类的初始化，构造与析构 - 构造函数执行顺序：基类、派生类成员类、派生类 - 析构函数执行次序相反 - 缺省执行基类默认构造函数，派生类成员初始化表可指定非默认构造函数

2. 虚函数

  • 虚函数、final与override

  • 纯虚函数和抽象类

  • 应用：抽象工厂

  • 不要重新定义虚函数继承而来的缺省参数值

    • 虚函数是动态绑定，而缺省参数是静态绑定。
    • 如果重新定义缺省参数值，程序会调用基类的虚函数。

3. 多继承

  • 基类声明次序决定了构造/析构函数的调用次序、数据成员的存储安排
  • 当发生成员名冲突，需要使用<基类名>::<基类成员名>
  • 如果直接基类有公共基类，那么公共基类中的成员在派生类中有多个副本
    • 虚基类（在继承公共基类的时候添加virtual关键字）可以使得公共基类在派生类中只有一个实例

C++ 虚函数、重载、模板

函数重载 - 名称相同、参数不同 - 静态绑定

操作符重载：提高可读性和可扩充性 - 方式：类成员函数（隐含this）/带有类参数的全局函数（友元） - = () [] 不能作为全局函数重载

  • 单目/双目、优先级、结合性
  • a++与++a

特殊操作符重载 - =，默认赋值操作符重载函数为逐个成员赋值，对含有对象成员的类递归进行。 - 赋值操作符重载不能继承 - 避免自我赋值 - () - [] - 类型转换：operator double() - ->

  • new和delete重载

void *operator new (size_t size, …)

void operator delete(void *p, size_t size)

重载后不再调用内置的new和delete。

  • 模板函数与模板类

new、delete与内存管理

new的过程 - 获得一块内存空间 - 调用构造函数 - 返回正确的指针

delete的过程 - 调用析构函数 - 确定指向分配空间的指针 - 将该空间归还系统

重载：使用链表构造内存池进行管理。
