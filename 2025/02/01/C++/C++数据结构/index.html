1. 内存

C++将内存视为一段连续单元，每个单元的大小为1字节且都有一个地址。通过其地址访问内存中的一个字节是一个\(\Omicron(1)\)常量时间操作。

然而，具体的计算机并非这样简单，现代CPU往往有多个核心，每个核心既有独属于自己的缓存，也有公用的缓存。

  • 以我当前使用的Macbook Pro 2021为例：可以用sysctl -a hw查看（下述KB均指1024B）

    M1芯片包含4个Firestrom高性能核心和4个Icestrom低性能核心

    • 每个Firestrom核心都有自己的192KB指令L1缓存和128KB数据L1缓存
    • 4个Firestrom核心共用12MB的L2缓存
    • 每个Icestrom核心都有自己的128KB指令L1缓存和64KB数据L1缓存
    • 4个Icestrom核心共用4MB的L2缓存
    • 每个缓存行的大小为128B，页大小16KB
    • 内存16GB

  • 一些数据：

    • L1缓存：1ns
    • L2缓存：5ns
    • 内存：100ns
    • 硬盘：15ms

2. 标准库容器

C++容器是一系列数据结构，封装了元素内存管理的细节，分为3类：序列容器、关联容器和容器适配器。

序列容器

标准库序列容器包含array、vector、deque、list、forward_list。此外也有处理字符的basic_string。

选择容器的思考：元素数量、使用方式（增删改查）、数据处理位置、元素排列顺序。

array

array使用栈而不是堆来管理元素，实际上是原本C数组的一个替代。

  • auto a = array<int, k>{};
  • 它使用类型和大小作为模板参数，这避免了C数组的数组衰变现象。
    • 数组衰变：当C数组作为参数传递给函数时，由于传递的是首地址指针，会丢失其大小信息。
    • 也可以用视图来解决数组衰变的问题。

vector

vector中的元素在堆内存中连续排列，因此可以用常数时间访问其中的任何元素。

vector具有大小和容量两个属性，大小指其中元素的个数，容量指实际具有的空间可以容纳的元素个数。vector有三种改变容量的方式：

  1. 使用push_back向vector中添加元素时，若不超出容量，就会直接添加；否则会申请一个新的空间，然后把所有元素移动到新空间。
    • 当我们用指针指向vector中的元素时，如果vector申请新的空间，那么原指针指向的位置就会无效，这种情况称为野指针，因此不要用指针指向vector中的元素。
    • vector在移动其中的元素时，如果该元素对象具有noexcept的移动构造函数，那么会使用移动构造，否则会使用复制构造。实际上，它使用move_if_noexcept。
    • 使用emplace_back可以直接在vector中创建对象，而不是先创建再复制或移动到vector中。
  2. 调用reserve，设定容量。
  3. 调用resize，且设定的大小大于当前容量。
  4. 调用shrink_to_fit，将容量削减为当前大小。

换言之，容量也就意味着vector的内存空间，不改变容量，也不会改变vector的内存空间地址。

  • clear清空vector，但不改变容量。
  • erase和erase_if删除指定元素和元素集合，但不改变容量，也就是说，删除后vector进行了元素的移动。

deque

在vector的开头添加或删除元素的时间复杂度为\(\Omicron(n)\)，在结尾有时候也不是\(\Omicron(1)\)，deque，也就是双端队列就是用于解决开头和结尾添加或删除的问题。

deque表现为一组固定大小的数组，因而可以以常数时间进行随机访问。然而它的所有元素并不是连续存储的。因而可以实现在开头和结尾以常数时间添加和删除元素。

随机位置插入和删除的复杂度为\(\Omicron(n)\)。

list

list是一个双向链表，而forward_list是一个单向链表。list因为具有后指针，会占用更多内存。

basic_string

string是basic_string<char>的一个typedef。

basic_string使用小对象优化，如果字符串的大小很小，就不会分配动态内存。

关联容器

关联容器根据元素本身特性放置元素。分为有序关联容器和无序关联容器：

  • 有序关联容器使用自平衡二叉搜索树来存储元素，要求元素按小于进行排序。
    • 增删改查的复杂度为\(\Omicron(\log n)\)。
    • set、map、multiset、multimap
  • 无序关联容器使用哈希表来存储元素，要求元素按==进行比较，并且可以计算哈希值。
    • 增删改查的复杂度为\(\Omicron(1)\)。
    • unordered_set、unordered_map、unordered_multiset、unordered_multimap

C++20后，可以使用contains函数来确认关联容器是否包含特定元素，不用使用count或find来确定。

容器适配器

stack、queue和priority_queue是标准库中的三种容器适配器，它们代表可以由底层序列容器实现的抽象数据类型。

视图

C++17引入了string_view和span，它们不是容器，而是连续元素的视图。

string_view使用指针和大小避免了复制，它的substr操作是常数复杂度，而string则需要申请空间并逐个赋值，是线性复杂度。

span是对于数组的视图，同样使用指针和大小避免复制。将函数参数设置为span而传入C数组，可以避免数组衰减。

3. 标准库算法

C++20引入了<ranges>，同时也改变了<algorithm>。过去我们使用算法都基于迭代器，例如sort(a.begin(),a.end())，现在可以直接使用ranges::sort(a)（并且这个函数在algorithm库中），对于其他算法也是如此。

算法概览

  1. 遍历：

    • for_each，对序列的每个元素调用一次参数中的函数，并且忽略函数返回值。

    • transform，对序列的每个元素调用一次参数中的函数，将函数返回值保存在第二个参数序列中。

    • generate，对序列的每个元素调用一次参数中的函数，将函数返回值保存在该当前元素中。

  2. 排序：sort

    • 使用is_sorted确定集合是排序的。

  3. 查找：

    • 线性：find
    • 二分：
      • binary_search
      • equal_range：参数的范围，返回一个pair
      • upper_bound：参数最后一个位置的迭代器
      • lower_bound：参数第一个位置的迭代器

  4. 条件测试：接受一个范围和一个判断函数

    • all_of
    • any_of
    • none_of

  5. 计算元素数量：count。

  6. 最小、最大：min、max；min_element、max_element。

  7. 夹逼：clamp，保证一个值在某个范围中。

  8. 交换：swap

  9. 反转序列：reverse

  10. 旋转序列：rotate，rotate(vec.begin(), vec.begin() + n, vec.end());

算法特性

  1. 不会改变容器大小
  2. 默认使用==和<运算符
  3. 复杂性保证：既不分配内存，也不具有高于\(\Omicron(n\log n)\)的时间复杂度。
